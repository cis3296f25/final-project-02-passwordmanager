This diagram shows how the password manager is built. The GUI (blue boxes) is what users see - login screen, main window, dialogs for adding/editing passwords. When you do something in the GUI, it sends HTTP requests through the API layer (red boxes) to the core logic (purple boxes), which handles encryption and saves everything to the database. We use an API internally so the GUI doesn't have to maintain logic about encryption or databases - it just makes requests and gets responses back. The entry point is in main.py which starts both the web server and the GUI.
